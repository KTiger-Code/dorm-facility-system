3.1  แดชบอร์ด Admin (/staff)

  รายละเอียดส่วนต่าง ๆ:

  ■ การ์ดสรุปงานค้าง
    - ใบแจ้งหนี้รอตรวจสลิป (payment_status = waiting_review)
    - คำขอซ่อมรอดำเนินการ (status = pending)
    - คำขอจองรออนุมัติ (status = pending)
    - รวมงานค้างทั้งหมด
    → คลิกที่การ์ดจะนำไปยังหน้าจัดการนั้น ๆ

  ■ Admin Logs
    - กดปุ่ม "แสดง Logs / ซ่อน Logs" เพื่อเปิด/ปิด
    - แสดงสถิติ: Logs วันนี้ / จำนวน Admin ใช้งาน (7 วัน)
    - สถิติ Action ยอดนิยม (สร้าง, แก้ไข, ลบ, อนุมัติ, ปฏิเสธ)
    - ตาราง Logs ล่าสุด: ชื่อ Admin, วันเวลา, Action, ตาราง, IP
    - แบ่งหน้าครั้งละ 10 รายการ

  ■ Quick Links
    - ลิงก์ไปยังเมนูจัดการต่าง ๆ (พัสดุ, ประกาศ, ผู้ใช้, ฯลฯ)

3.2  จัดการใบแจ้งหนี้ (/admin/invoice)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  3.2.1  การสร้างใบแจ้งหนี้ใหม่
  ────────────────────────────

  1. กดปุ่ม "+ สร้างใบแจ้งหนี้ใหม่"
  2. กรอกข้อมูล:

     ก. ข้อมูลห้อง
        - เลขห้อง: พิมพ์เลขห้อง ระบบตรวจสอบว่ามีอยู่จริง
          (Autocomplete จากรายชื่อผู้ใช้)
        - เดือน: เลือกเดือน (01-12)
        - ปี: เลือกปี (พ.ศ.)

     ข. ค่าน้ำ 💧
        - มิเตอร์ก่อนหน้า: ดึงอัตโนมัติจากใบแจ้งหนี้เดือนก่อน (readonly)
        - มิเตอร์ปัจจุบัน: กรอกตัวเลขมิเตอร์ใหม่
        - ราคาต่อหน่วย: กรอกราคา (บาท/หน่วย)
        → ค่าน้ำ = (ปัจจุบัน - ก่อนหน้า) × ราคาต่อหน่วย (คำนวณอัตโนมัติ)

     ค. ค่าไฟฟ้า ⚡
        - มิเตอร์ก่อนหน้า: ดึงอัตโนมัติ (readonly)
        - มิเตอร์ปัจจุบัน: กรอกตัวเลข
        - ราคาต่อหน่วย: กรอกราคา
        → ค่าไฟ = (ปัจจุบัน - ก่อนหน้า) × ราคาต่อหน่วย (คำนวณอัตโนมัติ)

     ง. ค่าใช้จ่ายอื่น
        - ค่าส่วนกลาง: กรอกจำนวนเงิน
        - ค่าเช่าห้อง: กรอกจำนวนเงิน

     จ. รายการเพิ่มเติม (Extra)
        - กดปุ่ม "+ เพิ่มรายการ"
        - กรอก ชื่อรายการ (เช่น ค่าอินเทอร์เน็ต) + จำนวนเงิน
        - เพิ่มได้หลายรายการ กดปุ่ม "ลบ" เพื่อนำออก

  3. กดปุ่ม "💾 บันทึก"
  4. ระบบบันทึกใบแจ้งหนี้ + รายการ Extra + Admin Log

  ⚠️ หมายเหตุ:
  • ระบบตรวจสอบว่าเลขห้องมีผู้เช่าหรือไม่
  • มิเตอร์ก่อนหน้าดึงจากใบแจ้งหนี้เดือนก่อนอัตโนมัติ
  • ค่าน้ำ/ค่าไฟคำนวณแบบ Real-time ขณะกรอก


  3.2.2  การจัดการรายการใบแจ้งหนี้
  ────────────────────────────────

  ■ ค้นหา & กรอง
    - ช่องค้นหา: พิมพ์เลขห้องเพื่อค้นหา
    - ตัวกรองสถานะ: ทั้งหมด / จ่ายแล้ว / รอตรวจสลิป / รอชำระ

  ■ ตารางใบแจ้งหนี้
    คอลัมน์: ห้อง | เดือน | ค่าน้ำ | ค่าไฟ | ค่าส่วนกลาง |
             ค่าเช่า | รายการเพิ่ม | รวม | สถานะ | สลิป | จัดการ

  ■ ปุ่มจัดการ (แต่ละแถว)
    - ✏️ แก้ไข    — โหลดข้อมูลเข้าฟอร์มเพื่อแก้ไข
    - 🗑️ ลบ      — ลบใบแจ้งหนี้ (ยืนยันก่อนลบ)
    - ✅ Checkbox — ทำเครื่องหมายจ่ายแล้ว/ยังไม่จ่าย


  3.2.3  การตรวจสอบสลิปชำระเงิน
  ──────────────────────────────

  1. ที่ใบแจ้งหนี้สถานะ "รอตรวจสลิป" จะเห็นรูปภาพสลิป
  2. คลิกที่รูปเพื่อดูภาพขนาดใหญ่
  3. ตรวจสอบความถูกต้องของยอดเงินและข้อมูล
  4. หากถูกต้อง: ทำเครื่องหมาย ✅ Paid = จ่ายแล้ว
     → สถานะเปลี่ยนเป็น "paid"
  5. หากไม่ถูกต้อง: ไม่ต้องทำอะไร หรือลบสลิปเพื่อให้ผู้พักอาศัยอัปโหลดใหม่

3.3  จัดการจองพื้นที่ (/admin/facility-booking)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  3.3.1  ภาพรวม
  ─────────────

  แสดงสถิติ: ทั้งหมด / รออนุมัติ / อนุมัติแล้ว / ถูกปฏิเสธ

  ตารางรายการจอง:
    ห้อง | สถานที่ | วันที่จอง | เวลา | จำนวนคน | สถานะ | จัดการ


  3.3.2  การอนุมัติ/ปฏิเสธ
  ────────────────────────

  สำหรับคำขอที่สถานะ "pending" (รออนุมัติ):

  1. ตรวจสอบรายละเอียดการจอง
  2. กดปุ่ม ✅ (อนุมัติ) หรือ ❌ (ปฏิเสธ)
  3. สถานะอัปเดตทันที
  4. ระบบบันทึก Admin Log

  เกณฑ์พิจารณา:
  • ซ้ำซ้อนกับการจองอื่นหรือไม่
  • จำนวนคนเหมาะสมกับสถานที่หรือไม่
  • เวลาจองสมเหตุสมผลหรือไม่


  3.3.3  การแก้ไขรายละเอียด (Inline Edit)
  ───────────────────────────────────────

  1. กดปุ่ม ✏️ ที่รายการที่ต้องการแก้ไข
  2. แถวเปลี่ยนเป็นโหมดแก้ไข สามารถแก้ได้:
     - ชื่อสถานที่/กิจกรรม
     - วันที่จอง
     - เวลาเริ่ม / เวลาสิ้นสุด
     - จำนวนผู้ใช้
  3. กดปุ่ม 💾 เพื่อบันทึก หรือ ❌ เพื่อยกเลิกการแก้ไข


  3.3.4  การลบรายการจอง
  ─────────────────────

  1. กดปุ่ม 🗑️ ที่รายการที่ต้องการลบ
  2. ยืนยันการลบ
  3. ระบบลบรายการและบันทึก Admin Log

3.4  จัดการแจ้งซ่อม (/admin/repairs)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  3.4.1  ภาพรวม
  ─────────────

  สถิติ: ทั้งหมด / รอดำเนินการ / กำลังดำเนินการ / เสร็จสิ้น
  ตัวกรอง: กรองตามสถานะ (ทั้งหมด, pending, in_progress, completed)


  3.4.2  การอัปเดตสถานะซ่อม
  ─────────────────────────

  1. ที่รายการคำขอซ่อม เลือก Dropdown สถานะ
  2. เลือกสถานะใหม่:

     สถานะที่เลือกได้      ความหมาย
     ───────────────────  ──────────────────
     pending              รอดำเนินการ
     in_progress          กำลังดำเนินการ
     completed            ซ่อมเสร็จเรียบร้อย

  3. สถานะอัปเดตทันทีที่เลือก
  4. ระบบบันทึก Admin Log

  ลำดับทั่วไป:  pending → in_progress → completed


  3.4.3  การลบคำขอซ่อม
  ─────────────────────

  1. กดปุ่ม 🗑️ ที่รายการที่ต้องการลบ
  2. ยืนยันการลบ
  3. ระบบลบรายการและบันทึก Admin Log

3.5  จัดการประกาศ (/admin/announcement)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  3.5.1  การสร้างประกาศใหม่
  ────────────────────────

  1. กรอกข้อมูล:
     - หัวข้อ (Title): สูงสุด 100 ตัวอักษร (มีตัวนับ)
     - เนื้อหา (Content): สูงสุด 500 ตัวอักษร (มีตัวนับ)
  2. ดูตัวอย่าง Preview ด้านข้าง (แสดงขณะพิมพ์)
  3. กดปุ่ม "📢 โพสต์ประกาศ"
  4. กดปุ่ม "🧹 ล้าง" เพื่อเคลียร์ฟอร์ม


  3.5.2  การแก้ไขประกาศ
  ─────────────────────

  1. ที่รายการประกาศ กดปุ่ม ✏️ แก้ไข
  2. ข้อมูลจะถูกโหลดเข้าฟอร์มด้านบน
  3. แก้ไขหัวข้อ/เนื้อหา แล้วกดปุ่ม "โพสต์" เพื่อบันทึก


  3.5.3  การลบประกาศ
  ──────────────────

  1. กดปุ่ม 🗑️ ที่ประกาศที่ต้องการลบ
  2. ยืนยันการลบ
  3. ระบบลบและบันทึก Admin Log


  3.5.4  การค้นหาประกาศ
  ─────────────────────

  • ช่องค้นหา: พิมพ์คำสำคัญจากหัวข้อหรือเนื้อหา
  • สถิติ: ทั้งหมด / ประกาศล่าสุด (7 วัน)

3.6  จัดการพัสดุ (/admin-parcels)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  3.6.1  การลงทะเบียนพัสดุใหม่
  ───────────────────────────

  1. กรอกข้อมูล:
     - เลขห้อง: พิมพ์เลขห้อง (ระบบตรวจสอบและ Autocomplete)
     - รายละเอียดพัสดุ: เช่น "กล่อง Shopee", "จาก Kerry"
  2. กดปุ่ม "📦 ลงทะเบียนพัสดุ + ส่ง LINE"
  3. ระบบดำเนินการ:
     ก. บันทึกข้อมูลพัสดุลง Database
     ข. ค้นหา line_user_id ของเจ้าของห้อง
     ค. หากมี LINE → ส่ง Flex Message แจ้งเตือนอัตโนมัติ
     ง. หากไม่มี LINE → แจ้งว่าส่งไม่ได้ (ผู้พักอาศัยยังไม่เชื่อมต่อ LINE)

  ผลลัพธ์ที่แสดง:
    ✅ "ลงทะเบียนสำเร็จ, LINE sent = true"   (ส่ง LINE ได้)
    ⚠️ "ลงทะเบียนสำเร็จ, LINE sent = false"  (ส่ง LINE ไม่ได้)


  3.6.2  การจัดการรายการพัสดุ
  ─────────────────────────

  ■ ค้นหา & กรอง:
    - ค้นหาตามเลขห้อง
    - กรองตามสถานะ: ทั้งหมด / รับแล้ว / ยังไม่รับ

  ■ สถิติ: ทั้งหมด / รับแล้ว / ยังไม่รับ

  ■ ตารางพัสดุ:
    ห้อง | รายละเอียด | ผู้รับ | สถานะ (Checkbox) | วันที่ | ลบ

  ■ การทำเครื่องหมายรับพัสดุ:
    - กดเครื่องหมาย ☑ ที่คอลัมน์ "สถานะ"
    - ระบบบันทึก picked_up = true + picked_up_at = วันเวลาปัจจุบัน
    - กดอีกครั้งเพื่อ Toggle กลับเป็นยังไม่รับ

  ■ การลบพัสดุ:
    - กดปุ่ม 🗑️ แล้วยืนยัน

3.7  จัดการผู้ใช้ (/admin-users)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  3.7.1  การเพิ่มผู้ใช้ใหม่
  ─────────────────────────

  1. กดปุ่ม "+ เพิ่มผู้ใช้ใหม่"
  2. กรอกข้อมูล:
     - Username: ชื่อผู้ใช้สำหรับ Login (ต้องไม่ซ้ำ)
     - Password: รหัสผ่าน (บังคับกรอกสำหรับผู้ใช้ใหม่)
     - ชื่อ-นามสกุล: ชื่อเต็ม
     - เลขห้อง: เช่น A101, B203
     - บทบาท: เลือก resident (ผู้เช่า) หรือ admin (ผู้ดูแล)
  3. กดปุ่ม "💾 บันทึก"
  4. ระบบสร้างบัญชีผู้ใช้ใหม่และบันทึก Admin Log


  3.7.2  การแก้ไขผู้ใช้
  ─────────────────────

  1. กดปุ่ม ✏️ ที่ผู้ใช้ที่ต้องการแก้ไข
  2. ข้อมูลโหลดเข้าฟอร์ม
  3. แก้ไขข้อมูลที่ต้องการ:
     - ชื่อผู้ใช้, ชื่อเต็ม, เลขห้อง, บทบาท
     - รหัสผ่าน: ปล่อยว่างหากไม่ต้องการเปลี่ยน
       หรือกรอกใหม่เพื่อรีเซ็ตรหัสให้ผู้ใช้
  4. กดปุ่ม "💾 บันทึก"


  3.7.3  การลบผู้ใช้
  ──────────────────

  1. กดปุ่ม 🗑️ ที่ผู้ใช้ที่ต้องการลบ
  2. ยืนยันการลบ
  3. ระบบลบบัญชีและบันทึก Admin Log

  ⚠️ คำเตือน: การลบผู้ใช้ที่มีใบแจ้งหนี้/คำขอซ่อม อาจกระทบข้อมูลที่เกี่ยวข้อง


  3.7.4  การค้นหาผู้ใช้
  ─────────────────────

  • ค้นหาด้วยชื่อหรือเลขห้อง
  • สถิติ: ทั้งหมด / ผู้เช่า / ผู้ดูแล

3.8  Admin Logs (ระบบ Audit Trail)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ระบบบันทึกทุกการดำเนินการของ Admin อัตโนมัติ

  ข้อมูลที่บันทึก:
  ─────────────────────────────────────────────
  ฟิลด์          ตัวอย่าง
  ─────────────  ──────────────────────────────
  admin_id       1 (ID ของ Admin ที่ดำเนินการ)
  action         CREATE, UPDATE, DELETE,
                 APPROVE_BOOKING, REJECT_BOOKING
  target_table   invoices, repair_requests,
                 facility_bookings, announcements,
                 parcels, users
  target_id      25 (ID ของ Record ที่ถูกแก้ไข)
  details        รายละเอียดเพิ่มเติม (JSON)
  ip_address     192.168.1.100
  created_at     2026-03-08 10:30:45

  การดำเนินการที่ถูกบันทึก:
  • สร้าง/แก้ไข/ลบใบแจ้งหนี้
  • อนุมัติ/ปฏิเสธการจองพื้นที่
  • อัปเดตสถานะแจ้งซ่อม
  • สร้าง/แก้ไข/ลบประกาศ
  • สร้าง/แก้ไข/ลบผู้ใช้

  การดู Admin Logs:
  → ที่หน้า Dashboard (/staff) กดปุ่ม "แสดง Logs"

  หมายเหตุ: Admin Logs ไม่สามารถลบได้ เพื่อความปลอดภัยและตรวจสอบย้อนหลัง
